# Идеи на будущее

Вторичный документ. Идеи по развитию — **не** задачи и **не** обещания. Часть взята
из [`ROADMAP.md`](../ROADMAP.md) (незакрытые пункты), часть — наблюдения из анализа.
Реализованное состояние проекта описано в остальных специфах; здесь — только «что
можно было бы».

## Из ROADMAP.md (незакрытые пункты)

| Пункт                                          | Заметка к реализации                                   |
|------------------------------------------------|--------------------------------------------------------|
| Изображения в карточках постов и в теле постов | Есть `static/assets/images/posts/` (пока `.gitkeep`). Потребуется поле во frontmatter (напр. `cover`) + отрисовка в `index.html`-карточке и `posts/single.html`. Помнить про `relativeURLs`. |
| Темы для code-блоков                           | Hugo Chroma (server-side, без JS). Настроить `[markup.highlight]` в `hugo.toml`, стили под dark/light через CSS-переменные. Без highlight.js (ADR-5). |
| Аналитика                                      | Выбрать приватный/легковесный вариант; вписать в `head.html`. Взвесить против «никаких сторонних скриптов». |
| Пагинация на MAIN и POSTS                      | Сейчас архив — единая страница с группировкой по годам. Hugo `.Paginate`. Затронет `posts/list.html`; продумать взаимодействие с per-language и `uglyURLs`. |
| Giscus (комментарии)                            | Сторонний виджет — противоречит духу ADR-5; требует явного решения владельца. |

## Принципы для любых будущих изменений

Любая идея отсюда должна пройти те же ворота, что и всякая фича:
- **Не нарушать ADR-5** (никаких фреймворков/бандлеров/сторонних библиотек без
  явной просьбы) — [`architecture/tech-stack.md`](architecture/tech-stack.md).
- **Двуязычность** — работать на ru и en ([`conventions/bilingual-model.md`](conventions/bilingual-model.md)).
- **Домен** — только в `static/CNAME` (ADR-4).
- **Обновлять специфы** в том же изменении.

См. [`guides/add-new-feature.md`](guides/add-new-feature.md).
