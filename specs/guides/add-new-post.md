# Руководство: добавить пост

> Каноничное пошаговое руководство по добавлению поста. При расхождении с кодом —
> верить коду.

## Бэкграунд

Пост — отдельный файл на язык с общим базовым именем (см.
[`conventions/bilingual-model.md`](../conventions/bilingual-model.md)). Может быть
двуязычным (норма) или одноязычным.

## Двуязычный пост (оба языка)

### 1. Выбрать слаг
ASCII, kebab-case, латиницей даже для русского поста — `my-new-post`. Это и URL, и
базовое имя файлов. См. [`conventions/naming.md`](../conventions/naming.md).

### 2. Скаффолдить оба файла из архетипа
```
hugo new posts/my-new-post.ru.md
hugo new posts/my-new-post.en.md
```
Архетип (`archetypes/posts.md`) даёт заготовку frontmatter + пример секции с якорем.

### 3. Заполнить frontmatter на каждом файле
```yaml
---
title: "Заголовок на языке файла"
date: 2026-01-15            # ОДИНАКОВАЯ на ru/en
description: "1–2 предложения — идут в карточку, мета-тег, поиск."
tags: [foo, bar]           # англ. слаги, ОДИНАКОВЫЕ на ru/en; можно []
---
```
Полная схема — [`data-model/frontmatter-reference.md`](../data-model/frontmatter-reference.md#пост).
**Не добавляйте `url:`** — плоский URL строится автоматически.

### 4. Написать тело на языке файла
- Первая строка — интро-абзац в `<p>...</p>` (plain-текст этого языка).
- Секции — `## Заголовок {#ascii-anchor}`, подсекции — `###`.
- Якорь **обязателен, ASCII, уникален в файле** (см.
  [`conventions/naming.md`](../conventions/naming.md#якоря-заголовков-) и ADR-6). ToC
  строится автоматически — руками TOC не пишите.

### 5. Проверить локально
`hugo server`, открыть:
- ru: `http://localhost:1313/posts/my-new-post.html`
- en: `http://localhost:1313/en/posts/my-new-post.html`

Убедиться, что переключатель языка на посте линкует между версиями.

### 6. Провалидировать перед публикацией
Делегировать субагенту **`blog-post-validator`** (через Agent tool) — он проверяет
парность языков, якоря, грамматику/пунктуацию, форматирование Markdown/кода. Скилл
`validate-blog-post` **не** вызывать напрямую из основного диалога.

## Одноязычный пост (только ru или только en)

То же, но создать **только один** файл (`.ru.md` **или** `.en.md`). Больше ничего:
- на другом языке поста просто нет (нет в списке/архиве/RSS/sitemap);
- переключатель языка через `.IsTranslated` уходит на главную того языка, не в 404;
- позже добавите второй файл с тем же слагом — Hugo спарит автоматически.

## Пост в серии

Добавить объект во frontmatter:
```yaml
series:
  name: "distributed-systems"   # = имя файла страницы серии
  number: 3
```
Лучше делегировать субагенту **`series-editor`** (скилл `add-post-to-series`) — он
создаст страницы серии при необходимости и присвоит номер. См.
[`components/series.md`](../components/series.md).

## Частые ошибки
| Симптом                              | Причина                                     |
|--------------------------------------|---------------------------------------------|
| Build падает на заголовке            | забыт `{#slug}` или он не-ASCII (ADR-6)      |
| Build падает «duplicate anchor»      | два одинаковых `{#slug}` в файле            |
| Переключатель ведёт на главную       | нет парного языкового файла / разное базовое имя |
| Пост не в поиске                     | пустой `description` (индексируется, но без описания) |

## Чек-лист
- [ ] слаг ASCII kebab-case, одинаковый для ru/en
- [ ] оба файла (или сознательно один)
- [ ] `date` одинаковая
- [ ] `description` заполнен на каждом языке
- [ ] интро-`<p>`, секции с якорями
- [ ] якоря ASCII и уникальны
- [ ] проверены обе версии + переключатель языка
- [ ] прогнан `blog-post-validator`
