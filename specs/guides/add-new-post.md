# Руководство: добавить пост

> Каноничное пошаговое руководство по добавлению поста. При расхождении с кодом —
> верить коду.

## Бэкграунд

Пост — отдельный файл на язык с общим базовым именем (см.
[`conventions/bilingual-model.md`](../conventions/bilingual-model.md)). Может быть
двуязычным (норма) или одноязычным.

## Двуязычный пост (оба языка)

### 1. Выбрать слаг и номер
Слаг — ASCII, kebab-case, **настоящий английский, не транслит** (даже если пост
только на русском) — `my-new-post`, а не `moy-novyy-post`. Читателю, не знающему
русский, транслит-URL совершенно бесполезен, а английский слаг понятен всем. Это
и есть URL поста (через `slug:` во frontmatter, см. шаг 3). См.
[`conventions/naming.md`](../conventions/naming.md#слаг--всегда-настоящий-английский-не-транслит).

Номер — следующий по счёту двузначный префикс (`ls content/posts/` — берите
`max + 1`, zero-padded: `07`, не `7`). Это только для сортировки директорий в
файловой системе, в URL не попадает.

### 2. Скаффолдить директорию поста и оба файла из архетипа
```
mkdir content/posts/07-my-new-post
hugo new posts/07-my-new-post/index.ru.md
hugo new posts/07-my-new-post/index.en.md
```
Архетип (`archetypes/posts.md`) даёт заготовку frontmatter + пример секции с якорем.
**Имя файла обязано быть ровно `index.ru.md`/`index.en.md`** — это Hugo page
bundle (см. [`data-model/content-organization.md`](../data-model/content-organization.md)),
любое другое имя не распознаётся как контент поста.

### 3. Заполнить frontmatter на каждом файле
```yaml
---
title: "Заголовок на языке файла"
date: 2026-01-15            # ОДИНАКОВАЯ на ru/en
description: "1–2 предложения — идут в карточку, мета-тег, поиск."
tags: [foo, bar]           # англ. слаги, ОДИНАКОВЫЕ на ru/en; можно []
slug: "my-new-post"        # ОБЯЗАТЕЛЬНО, ОДИНАКОВЫЙ на ru/en — без него URL включит "07-"
---
```
Полная схема — [`data-model/frontmatter-reference.md`](../data-model/frontmatter-reference.md#пост).
**Не добавляйте `url:`** — плоский URL строится автоматически из `slug`.

Опционально можно добавить обложку поста (показывается только на странице самого
поста, до заголовка; карточки на главной картинок не показывают):
```yaml
cover:                        # опционально, объект
  src: "images/cover.jpg"    # bundle-относительный путь, ОДИНАКОВЫЙ на ru/en
  alt: "Описание картинки на языке файла"           # переводится
```
Файлы кладите в `content/posts/07-my-new-post/images/` — **рядом с постом**, не в
`static/` (ресайза на сборке нет — готовьте уже нужного размера). Подробности —
[`components/posts.md`](../components/posts.md#обложка-cover-и-инлайн-картинки).

### 4. Написать тело на языке файла
- Первая строка — интро-абзац в `<p>...</p>` (plain-текст этого языка).
- Секции — `## Заголовок {#ascii-anchor}`, подсекции — `###`.
- Якорь **обязателен, ASCII, уникален в файле** (см.
  [`conventions/naming.md`](../conventions/naming.md#якоря-заголовков-) и ADR-6). ToC
  строится автоматически — руками TOC не пишите.

### 5. Проверить локально
`hugo server`, открыть:
- ru: `http://localhost:1313/posts/my-new-post.html`
- en: `http://localhost:1313/en/posts/my-new-post.html`

Убедиться, что переключатель языка на посте линкует между версиями.

### 6. Провалидировать перед публикацией
Делегировать субагенту **`blog-post-validator`** (через Agent tool) — он проверяет
парность языков, якоря, грамматику/пунктуацию, форматирование Markdown/кода. Скилл
`validate-blog-post` **не** вызывать напрямую из основного диалога.

## Одноязычный пост (только ru или только en)

То же, но создать **только один** файл в директории поста (`index.ru.md`
**или** `index.en.md`). Больше ничего:
- на другом языке поста просто нет (нет в списке/архиве/RSS/sitemap);
- переключатель языка через `.IsTranslated` уходит на главную того языка, не в 404;
- позже добавите второй файл в ту же директорию — Hugo спарит автоматически.

## Пост в серии

Добавить объект во frontmatter:
```yaml
series:
  name: "distributed-systems"   # = имя файла страницы серии
  number: 3
```
Лучше делегировать субагенту **`series-editor`** (скилл `add-post-to-series`) — он
создаст страницы серии при необходимости и присвоит номер. См.
[`components/series.md`](../components/series.md).

## Частые ошибки
| Симптом                              | Причина                                     |
|--------------------------------------|---------------------------------------------|
| Build падает на заголовке            | забыт `{#slug}` или он не-ASCII (ADR-6)      |
| Build падает «duplicate anchor»      | два одинаковых `{#slug}` в файле            |
| Переключатель ведёт на главную       | нет парного языкового файла / файлы не в одной директории поста |
| URL включает номер (`/posts/07-my-new-post.html`) | забыт `slug:` во frontmatter — без него URL берётся из имени директории целиком |
| Пост не в поиске                     | пустой `description` (индексируется, но без описания) |
| Build падает «missing src or alt»    | у `cover` не хватает подполя, либо это не объект `{src, alt}` (например, строка) |
| Картинка не грузится (404), но build проходит без ошибок | файл не лежит в `images/` внутри директории поста, путь в `cover.src`/Markdown не совпадает с именем файла, или файл не закоммичен (это отдельные, не-текстовые файлы — легко забыть при `git add`); `Resources.GetMatch`/Markdown-рендер молча не находят файл, ошибки сборки не будет |
| Контент-файл не подхватывается Hugo вообще | назван не `index.ru.md`/`index.en.md` — любое другое имя внутри директории поста не распознаётся как page bundle |

## Чек-лист
- [ ] слаг — настоящий английский (не транслит), kebab-case, одинаковый для ru/en, задан в `slug:`
- [ ] директория поста названа `<NN>-<slug>` (номер — только для сортировки, следующий по счёту)
- [ ] контент-файлы названы ровно `index.ru.md`/`index.en.md` (или один из них)
- [ ] `date` одинаковая
- [ ] `description` заполнен на каждом языке
- [ ] интро-`<p>`, секции с якорями
- [ ] якоря ASCII и уникальны
- [ ] если есть `cover`/инлайн-картинки — лежат в `images/` внутри директории поста и закоммичены
- [ ] проверены обе версии + переключатель языка
- [ ] прогнан `blog-post-validator`
